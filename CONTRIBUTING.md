# Contributing to OS-ONE

OS-ONE is not owned. It is tended.

Anyone may contribute. You do not need to write code. A sharp question is as valuable as a pull request.

---

## Before You Start

1. Read the [`README.md`](README.md) — especially the Core Principles
2. Read the [`MANIFESTO.md`](MANIFESTO.md) — understand the philosophy
3. Check existing [Issues](../../issues) and [Discussions](../../discussions) before opening new ones

---

## Ways to Contribute

| Contribution | Path |
|---|---|
| Question / idea | Open a GitHub Discussion |
| Spec error or gap | Open an Issue with label `spec-error` |
| New spec section | Open an RFC in `rfcs/` |
| Reference implementation | PR to `prototypes/` |
| Research / prior art | PR to `research/` |
| Philosophical question | PR to `discussions/` |

---

## RFC Process

Major changes to the spec go through the RFC process. See [`rfcs/README.md`](rfcs/README.md).

For small corrections (typos, clarifications), a direct PR is fine.

---

## Pull Request Guidelines

- One concern per PR
- PR title: `[L2] Add memory eviction policy` — include the layer or section
- Link to the relevant issue or RFC
- Write in plain language — the audience includes non-engineers

---

## Code of Conduct

Be direct. Be honest. Be kind. Attack ideas, not people.

This is a project about the relationship between humans and AI. Treat everyone in this community accordingly.
